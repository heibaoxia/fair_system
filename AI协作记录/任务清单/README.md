# 任务清单

> [!tip]
> 后续任务不再集中写进一个总文件，而是按照“日期 + 第n轮任务”的方式单独建档。

## 命名规则

- `YYYY-MM-DD-第1轮任务.md`
- `YYYY-MM-DD-第2轮任务.md`
- 同一天如果继续发布新一轮，就继续递增轮次编号

## 标准结构

每一轮任务文件固定包含：

1. 这轮要解决的问题
2. 任务列表
3. 每个任务的 AI 执行指令
4. 每个任务的验收测试
5. 本轮完成标准

# T9 手动验收指南

在开始验收前，先运行下面两个命令，确保数据是最新的：

```bash
python seed_demo_data.py
python acceptance_helper.py
```

页面入口：

- 评分期进行中项目：`http://127.0.0.1:8000/project/2`
- 评分期已结束项目：`http://127.0.0.1:8000/project/3`
- 历史负载观察项目：`http://127.0.0.1:8000/project/4`

当前固定成员 ID：

- `3`：刘峰
- `4`：陈雨
- `5`：赵宁
- `6`：孙悦
- `7`：周航

当前项目创建者：

- 项目 `2` 创建者：`3`
- 项目 `3` 创建者：`5`
- 项目 `4` 创建者：`7`

## 先做的准备

1. 打开项目详情页后，在左下角把当前视角切到“项目创建者”
2. 刷新页面，确认顶部能看到 `⚡ 一键公平分配`
3. 如果看不到按钮，先检查是不是切错了成员视角

## 验收 1：评分期内禁止一键分配与手动拖拽

项目：`http://127.0.0.1:8000/project/2`

重点模块：

- `30`：评分提醒消息中心
- `31`：综合分导出报表

操作：

1. 用创建者 `3` 视角进入项目 `2`
2. 看顶部 `⚡ 一键公平分配` 按钮
3. 尝试把任一成员卡片拖到模块 `30` 或 `31`

预期：

- 一键分配按钮是灰色禁用状态
- 按钮提示是 `请等待评分结束`
- 模块不会出现可用的 drop 高亮反馈
- 拖拽时弹出 toast：`评分期内禁止手动分配`

## 验收 2：点击按钮后出现预分配弹窗

项目：`http://127.0.0.1:8000/project/3`

重点模块：

- `36`：供应商对账导入
- `37`：项目毛利看板

操作：

1. 用创建者 `5` 视角进入项目 `3`
2. 点击顶部 `⚡ 一键公平分配`

预期：

- 出现“公平分配预览”弹窗
- 弹窗中展示每位成员的：
  - `30天已有分数`
  - `本次新增分数`
  - `合计总分`
  - `模块列表`
- 顶部能看到 `公平指数`
- 底部有 `直接开工` 和 `先微调`

## 验收 3：直接开工

操作：

1. 在项目 `3` 的预览弹窗里点击 `直接开工`
2. 等页面刷新

预期：

- 模块 `36`、`37` 不再是 `待分配`
- 它们会进入 `开发中`
- 负责人会更新为预览方案中的成员

备注：

- 如果你想重复做这个测试，重新执行 `python seed_demo_data.py`

## 验收 4：先微调

操作：

1. 重新执行 `python seed_demo_data.py`
2. 回到项目 `3`
3. 再次点击 `⚡ 一键公平分配`
4. 点击 `先微调`

预期：

- 弹窗关闭
- 顶部出现提示条：`微调模式：拖拽模块卡片到目标成员完成调整。完成后点击“确认开工”。`
- 顶部同时出现 `确认开工` 按钮

## 验收 5：微调模式下拖拽调整

操作：

1. 在项目 `3` 中，把模块卡片 `36` 或 `37` 拖到另一个成员卡片上
2. 观察模块卡片负责人文案变化
3. 点击顶部 `确认开工`

预期：

- 拖拽后模块负责人立即变成新的目标成员
- 点击 `确认开工` 后页面刷新
- 刷新后模块状态变为 `开发中`
- 刷新后负责人仍是你刚刚调整后的成员

## 验收 6：没有待分配模块时的友好提示

操作：

1. 在项目 `3` 完成一次 `直接开工` 或 `确认开工`
2. 页面刷新后再次观察顶部按钮

预期：

- `⚡ 一键公平分配` 按钮变灰
- 提示为 `没有待分配模块`

## 建议的验收顺序

1. 先跑 `python seed_demo_data.py`
2. 验收项目 `2` 的评分期禁用
3. 验收项目 `3` 的预览弹窗
4. 验收项目 `3` 的 `先微调`
5. 再重新跑一次 `python seed_demo_data.py`
6. 验收项目 `3` 的 `直接开工`

## 如果验收中途状态乱了

直接执行：

```bash
python seed_demo_data.py
python acceptance_helper.py
```

这会把验收数据恢复到初始状态，并重新打印项目、成员、模块 ID。

# 项目中心化网页结构与 Canvas 规则设计

## 1. 总体方向

- 首页采用 **项目中心化**。
- 整体结构采用 **项目主线 + 个人资产侧挂**。
- 允许新增页面，但新页面必须与现有结构建立明确关系，不能形成平行体系。

## 2. 核心页面骨架

### 2.1 首页

首页只保留项目主线入口：

- 我的项目列表
- 项目广场入口
- 发布项目
- 待办提醒
- 招募/申请提醒

### 2.2 项目广场

项目广场用于个人找项目，承担：

- 搜索项目
- 筛选招募项目
- 查看项目招募详情
- 提交申请加入

公开招募时，平台优先向 **技能需求匹配** 的用户推送项目。

### 2.3 发布项目页

- 任何用户都可以发布项目。
- 谁发布项目，谁自动成为该项目 PM。
- 发布项目时可设置：
  - 项目名称
  - 项目简介
  - 项目模式（PM 拉人 / 公开招募 / 两者结合）
  - 业主设置
  - 评分维度
  - 模块初始结构

### 2.4 项目详情页

项目详情页是系统核心聚合页，包含：

- 项目概览
- 模块列表
- 成员列表
- 招募管理
- 模块需求技能
- 模块验收状态
- 模块结算状态
- 项目复盘入口或复盘结果
- 项目治理（如解散项目）

### 2.5 人物详情页

人物详情页作为个人资产页，包含：

- 基础信息
- 社交按钮
- 技能得分
- 项目经历
- 作品集
- 收入统计
- 主页公开范围设置

### 2.6 技能库

技能库不放首页，不做一级入口，仅作为被调起能力出现：

- 设置模块需求技能时调起
- 查看技能详情时调起
- 新建技能时调起

### 2.7 项目复盘页

- 不挂首页。
- 路径为：首页 -> 项目列表 -> 具体项目 -> 项目复盘。
- 项目处于复盘阶段时进入复盘页。
- 项目已结束时，在项目详情页直接可见复盘信息。

## 3. 关键业务规则

### 3.1 项目组人

项目组人有两种模式：

- PM 自己拉人
- 发布招募

两种模式允许结合使用：PM 先拉人，剩余名额再公开招募。

### 3.2 申请加入

申请加入流程：

- 用户输入申请消息（可留空）
- 提交申请加入
- PM 收到提醒
- PM 点开申请信息可查看：
  - 名字
  - ID
  - 头像
  - 性别
  - 申请消息
  - 与申请模块匹配的技能得分
- 若无匹配技能分，显示“暂无该技能评分”
- 点击申请人名字，进入人物详情页

### 3.3 人物详情页公开范围

人物详情页采用 **整页统一一套公开范围**，可多选，按最大范围理解：

- 仅自己
- 对好友
- 对所有人
- 申请时对 PM
- 对工作组成员

### 3.4 业主与验收

- PM 创建项目时可设置业主。
- 业主可以是 PM 自己，也可以像拉人一样指定/邀请他人。
- 模块验收链路为：
  - PM 验收
  - 业主确认
- 当 PM 与业主为同一人时，验收流程简化为单人通过。

### 3.5 付款与结算

付款模式采用 **阶段 + 模块融合**：

- 每个付款阶段包含若干模块
- 阶段序列中最后一个模块的成果，作为该阶段验收成果

资金流规则：

- 业主可按阶段付款
- 平台先托管项目款
- 模块款到账后先不发给个人
- 项目整体结束后，由业主确认再分配给个人

项目详情中不单独拆“资金阶段页”，统一内置为：

- 项目收入详情
- 模块完成状态（已完成 / 未完成）
- 模块结算状态（已结算 / 待结算）

### 3.6 项目中途解散

PM 中途解散项目时，必须满足以下之一：

- 全体成员同意
- 完成补偿/结算后再推进解散

补偿规则：

- 未开始模块：支付预计报酬的 20%
- 已进行模块：支付预计报酬的 80% 以上
- 已完成但未结算模块：应直接进入待结算/结算

建议在项目详情页中设置：

- 解散项目入口
- 解散项目弹窗
- 成员补偿清单
- 解散确认结果

## 4. 技能库与技能映射

### 4.1 平台技能库

平台维护技能库，覆盖当前各工种主流工作技能。

用户也可以自定义技能，填写：

- 技能名称
- 技能描述

平台在新建技能时，需要校验：

- 名称是否重复
- 名称不同但描述是否高度相似

### 4.2 模块需求技能

PM 创建项目模块时，可通过输入技能名称或描述设置需求技能。

平台行为：

- 自动匹配现有技能
- 若匹配结果不满意，PM 可自定义新技能
- 新技能建成后自动加入平台技能池

### 4.3 项目复盘与个人技能分

项目复盘时，技能综合分来源为：

- 组内评估 70%
- 业主评估 30%

其中 PM 计入组内评估。

技能映射规则：

- 模块完成综合分与技能单项分按 5:5 合成模块技能得分
- 示例：
  - 模块综合分 = 7
  - 审美单项分 = 5
  - 绘图单项分 = 8
  - 则：
    - 审美 = 7 × 0.5 + 5 × 0.5
    - 绘图 = 7 × 0.5 + 8 × 0.5

个人单一技能最终分：

- 该技能总得分 / 该技能被打分次数

人物详情页中，鼠标悬停在技能得分上时，应显示：

- 组员打分
- 业主打分
- 单项技能分数来源

## 5. Canvas 绘图规则

### 5.1 节点颜色

- 红色：完整页面
- 绿色：按钮 / 菜单 / 入口
- 黄色：内容块
- 蓝色：弹出页 / 弹窗流程
- 紫色：详情页
- 白色：说明性内容
- 橙色：规则 / 状态 / 结算规则
- 灰色：系统自动处理 / 自动计算 / 自动推送

### 5.2 连线颜色与语义

- 默认箭头线：页面跳转 / 业务推进 / 手动操作触发
- 紫色无箭头线：包含 / 归属 / 内置关系
- 橙色箭头线：系统自动触发 / 自动计算 / 自动推送

说明：

- 不使用虚线表达系统自动行为，统一改用 **橙色箭头线**。

### 5.3 节点命名规范

建议所有重要节点都加括号说明：

- 页面节点：`项目复盘页(仅复盘阶段/已结束项目可见)`
- 按钮节点：`申请加入(可填写申请消息)`
- 规则节点：`解散项目(未开始20%，进行中80%+)`
- 自动节点：`技能自动匹配(名称重复/描述相似校验)`

## 6. 推荐新增节点类型

建议新增以下结构节点：

- 项目广场(搜索/筛选公开招募项目)
- 发布项目页(任何用户可发布，发布者自动成为 PM)
- 项目招募详情(展示招募岗位/缺口/需求技能)
- 申请加入(可填写申请消息)
- 申请详情(PM 查看申请人信息与技能匹配)
- 人物详情页(技能分/项目经历/作品集/收入统计)
- 项目复盘页(组内70% + 业主30%)
- 项目收入详情(内置于项目详情)
- 解散项目弹窗(补偿与确认清单)
- 技能自动匹配(名称/描述匹配)
- 技能去重校验(名称相同/描述相似)
- 招募优先推送(优先推送给技能匹配用户)

## 7. 推荐新增连线关系

- 主页 -> 项目广场
- 主页 -> 发布项目页
- 项目广场 -> 项目招募详情
- 项目招募详情 -> 申请加入
- 申请加入 -> PM提醒
- PM提醒 -> 申请详情
- 申请详情 -> 人物详情页
- 项目详情页 -> 项目复盘页
- 项目详情页 -> 人物详情页
- 设置需求技能 -> 技能自动匹配
- 技能自动匹配 -> 技能去重校验
- 公开招募 -> 招募优先推送
- 项目复盘页 -> 技能映射到人物详情

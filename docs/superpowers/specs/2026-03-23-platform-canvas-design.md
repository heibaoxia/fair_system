# 平台化网页结构与模块化架构设计

## 目标

将现有“内部工评协作软件”升级为“平台软件”，同时兼容：

- 有需求的人发布项目、找到合适技能的人
- 有能力的人在平台接项目、参与协作
- 圈子/公司内部监控项目进展
- 平台保障验收、结算、放款，降低拖欠风险
- 以按劳分配为主要分配逻辑，同时兼容多种生产关系

## 产品定位

采用 `轻广场 + 强协作区` 的双域分层方案：

- `公开平台区`：负责找人、找项目、建立联系
- `协作工作区`：负责推进项目、交付成果、完成验收
- `保障中心`：负责托管、结算、放款、争议冻结
- `平台运营区`：负责审核、风控、组织级管理

## 页面重组原则

- `主页` 拆为 `平台首页` 与 `个人工作台`
- `项目列表页` 拆为 `项目广场`、`我的项目`、`我的申请/邀约`
- `项目详情页` 升级为 `项目工作台`，并挂接监控、保障、文件、验收子流程
- `社交中心` 降级为辅助域，不再承担平台主入口
- `人物详情页` 升级为正式 `个人主页`

## 对话与文件协作设计

### 对话

- 好友之间支持私聊对话
- 项目成员之间支持项目会话
- 对话文件属于沟通附件，不等同于正式交付成果

### 项目共享文件夹

平台为每个项目提供共享云端文件空间：

- 项目下按模块划分文件夹
- 成员只能修改自己负责模块文件夹内容
- 其他成员可预览、下载其他模块的阶段成果
- 模块一旦通过验收，对应模块文件夹锁定为只读
- 若模块返工，必须通过显式“重新开启模块”流程解锁

### 文件分层

文件必须分为两类：

- `项目共享文件`：用于协作、草稿、参考资料、阶段成果
- `正式交付成果`：用于验收、结算、放款依据

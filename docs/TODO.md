# 项目管理与模块流转核心功能任务清单

## 1. 数据库模型更新
- [ ] 修改 `ModuleAssessment` 模型中的打分维度为：难度 (difficulty_score)、预计时间 (estimated_hours)、枯燥程度 (boredom_score)、工作强度 (intensity_score)。
- [ ] 确保 `Module` 模型中包含明确的：名称 (name)、描述 (description)、成果要求 (allowed_file_types)、完成状态 (status)。因为在不同项目同名模块本质不同，主键已保证了唯一性。

## 2. Pydantic Schemas 及 API 接口更新
- [ ] 更新 `schemas.py` 里的 `ModuleAssessment` 及相关创建类的字段匹配新模型。
- [ ] 更新 `projects.py` 增加更新项目（编辑已有项目）的代码接口。
- [ ] 更新获取所有项目的接口，以便前端选择编辑。
- [ ] 更新 `assignments.py` 中的规则引擎（或者保留后续再优化引擎内部计算逻辑，目前只需适配新传入的四个打分维度）。

## 3. 前端界面：项目管理总窗口
- [ ] 在首页顶部放置【📁 项目管理】大按钮。
- [ ] 弹窗内支持【新建项目】和【编辑已有项目】两种模式切换。
- [ ] 弹窗内包含基本的项目信息输入；以及“预设模块快速选择”功能页。

## 4. 前端界面：模块定义与节点连线工作流（V1初版）
- [ ] 支持动态添加和定义新模块（名称/描述/成果要求/状态）。
- [ ] 提供一个“工作流画板”区域，将定义好的模块卡片变成可拖拽的“节点”。
- [ ] 初步实现节点之间的前后放置或连线，以生成模块的前置/后置依赖关系（存储入 `FileDependency` 表）。
- [ ] 将项目、模块、依赖流一起打包保存给后端接口。

## 📚 关键概念记录

### FastAPI 核心概念
1. **路由（Route）**：URL路径与处理函数的映射
2. **装饰器（Decorator）**：`@app.get()`、`@app.post()` 等
3. **请求参数**：
   - 路径参数：`/projects/{id}`
   - 查询参数：`?skip=0&limit=100`
   - 请求体：POST/PUT 提交的数据
4. **响应模型**：使用 Pydantic 定义返回数据结构
5. **依赖注入**：自动管理数据库会话等资源

### Pydantic 数据验证
- 基于 Python 类型提示
- 自动验证输入数据格式
- 支持嵌套模型和复杂验证规则

### SQLAlchemy ORM
- 对象关系映射：Python类 ↔ 数据库表
- 会话（Session）管理数据库连接
- 查询API：`db.query(Model).filter(...)`

## 🔗 models.py 与 schemas.py 关系分析

### 1. 设计哲学对比

**models.py (SQLAlchemy)** - **数据库视角**
- 关注数据如何存储在数据库中
- 定义表结构、字段类型、约束、索引
- 建立表之间的外键关系
- 面向数据库操作（CRUD）

**schemas.py (Pydantic)** - **API视角**
- 关注数据如何在API中传输
- 定义请求/响应的数据格式
- 验证用户输入的数据
- 面向API交互

### 2. 字段映射关系

#### Member 模型对比
| models.py (数据库字段) | schemas.py (API字段) | 说明 |
|------------------------|----------------------|------|
| `id` (Integer, PK) | `id: int` | 数据库主键 → API响应字段 |
| `name` (String) | `name: str` | 直接映射 |
| `email` (String, unique) | `email: str` | 直接映射 |
| `is_active` (Boolean) | `is_active: bool` | 直接映射 |
| `projects` (relationship) | `projects: List[Project]` | 关联对象列表 |

**注意**：schemas.py 中 **缺少 Member 模型**，需要补充

#### Project 模型对比
| models.py | schemas.py | 说明 |
|-----------|------------|------|
| `id` | `id: int` | 主键映射 |
| `name` | `name: str` | 直接映射 |
| `description` | `description: Optional[str]` | 可选字段 |
| `created_by` (ForeignKey) | ❌ 未直接暴露 | 外键在API中可能隐藏 |
| `created_at` | ❌ 未包含 | 时间戳通常自动生成 |
| `modules` (relationship) | `modules: List[Module]` | 关联模块列表 |

#### Module 模型对比
| models.py | schemas.py | 说明 |
|-----------|------------|------|
| `id` | `id: int` | 主键映射 |
| `name` | `name: str` | 直接映射 |
| `estimated_hours` | `estimated_hours: float = 0.0` | 带默认值 |
| `project_id` (ForeignKey) | `project_id: int` | 外键作为普通字段 |

### 3. 关键设计差异

#### 字段可见性
- **models.py**：包含所有数据库字段（包括外键、时间戳）
- **schemas.py**：只暴露API需要的字段
  - 创建时：用户提供的字段（如 `name`, `description`）
  - 响应时：用户关心的字段（如 `id`, `modules` 列表）

#### 关联关系的处理
**models.py**：
```python
# 双向关联
projects = relationship("Project", back_populates="owner")
modules = relationship("Module", back_populates="project")
```

**schemas.py**：
```python
# 嵌套对象
modules: List[Module] = []
projects: List[Project] = []  # 需要补充
```

**区别**：
- SQLAlchemy：通过外键和relationship建立数据库级关联
- Pydantic：通过嵌套模型建立API响应结构

#### 默认值处理
**models.py**：数据库默认值
```python
is_active = Column(Boolean, default=True)
created_at = Column(DateTime, default=datetime.now)
estimated_hours = Column(Float, default=0)
```

**schemas.py**：API默认值
```python
estimated_hours: float = 0.0
is_active: bool = True  # 需要补充
```

### 4. 数据转换流程

```
API请求 JSON
    ↓
schemas.ProjectCreate (验证)
    ↓
提取数据 → 业务逻辑
    ↓
models.Project (数据库操作)
    ↓
SQLAlchemy对象 (查询结果)
    ↓
schemas.Project (转换，使用 from_attributes=True)
    ↓
API响应 JSON
```

### 5. 发现的问题与建议

#### 问题1：Member 模型缺失
- schemas.py 缺少 Member 相关模型
- 影响：无法实现成员管理API

**建议补充**：
```python
# 在 schemas.py 中添加
class MemberBase(BaseModel):
    name: str
    email: str

class MemberCreate(MemberBase):
    is_active: bool = True

class Member(MemberBase):
    id: int
    is_active: bool
    projects: List[Project] = []

    class Config:
        from_attributes = True
```

#### 问题2：外键字段暴露
- `Project.created_by` 是外键，但API中可能需要关联对象
- 选择：暴露ID还是关联对象？

**常见做法**：
```python
# 方案A：只暴露ID（简单）
created_by: int

# 方案B：暴露关联对象（完整）
owner: Member
```

#### 问题3：时间字段处理
- `created_at` 在 models.py 但不在 schemas.py
- 通常：创建时自动生成，响应时返回

**建议**：
```python
# 在 Project 模型中添加
created_at: datetime
```

### 6. 实际应用示例

假设创建项目：
```python
# 用户请求
POST /projects
{
    "name": "新项目",
    "description": "项目描述"
}

# schemas.ProjectCreate 验证通过
# 转换为 models.Project 对象
db_project = Project(
    name="新项目",
    description="项目描述",
    created_by=1,  # 当前用户ID
    created_at=datetime.now()
)

# 保存到数据库
db.add(db_project)
db.commit()

# 查询返回（包含关联模块）
db_project = db.query(Project).filter(Project.id==1).first()

# 转换为 schemas.Project 响应
response = Project.model_validate(db_project)
# 返回给用户
```

### 7. 学习要点

1. **分离关注点**：数据库结构 vs API数据格式
2. **字段映射**：不是所有数据库字段都需要暴露
3. **关联处理**：SQLAlchemy外键 ↔ Pydantic嵌套对象
4. **数据转换**：`from_attributes=True` 是关键桥梁
5. **设计决策**：哪些字段暴露、如何关联、默认值处理
